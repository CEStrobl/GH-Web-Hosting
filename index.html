GitHub Web Hosting Tutorial

in only 8 steps!

itty bitty things to notice:

  • idk if everything is going to work on mac ive never tested it but you can try it anyway if u want
  • this isnt a great method if you want to continually update your site. its more of just if you want to get it up and running quickly.
  • if you do want to update your site often, i recommend installing the github desktop app and watching a youtube video on setting it up. the desktop app will sync live changes but i personally find it to be kinda confusing. i'll try to make a tutorial on that one day...




Step 1

Make an account on github.com (if you don't already have one)

Step 2

Click the 'New' button

Step 3

Type in a repository name (your website name) and check the 'Add a README file' box then click 'Create repository'

Step 4

Now click 'Add file' and click 'Upload files'

Step 5

Add all the files in your website folder on your desktop to github and then click 'Commit changes'


*** GitHub looks for the 'index.html' file outside of folders. So add all the individual files in your main website folder. If you upload just the main folder, even if it has all the website files that work perfectly, it will not show up in the live site.

Step 6

Yayayayay we have our files but now we have to get the link to the web page.... click on the 'Settings' tab

Step 7

Keep scrolling down until you see the 'GitHub Pages' section and then click 'None' and select the 'main' branch

Step 8

Once you click 'Save' and scroll back down to 'GitHub Pages' then the link shows up and you can visit your lovely website


*** It may take up to 20 mins to show up tho
